# 1.2 安装与启动

## 环境要求

| 依赖 | 要求 |
|------|------|
| Python | 3.12 或更高版本 |
| Shell | macOS / Linux 推荐使用 `bash` 或 `zsh` |
| 依赖安装 | `start.sh` 优先使用 `uv`，未安装时回退到 `python3 -m venv` 与 `pip` |

Windows 用户可在 WSL、Git Bash 或其他兼容 Bash 的环境中运行启动脚本。

## 获取代码

```bash
git clone git@github.com:ahtcfg24/codex-shift.git
cd codex-shift
```

如果你已经在本地有仓库，直接进入项目目录即可。

## 准备环境变量

复制环境变量示例文件：

```bash
cp .env.example .env
$EDITOR .env
```

`.env` 中的变量名需要和 `config.yaml` 中 provider 的 `api_key_env` 一致。例如：

```dotenv
DEEPSEEK_API_KEY=sk-...
MIMO_TOKEN_API_KEY=...
```

`.env` 已被 `.gitignore` 排除，请不要提交真实 API Key。

如果 `apiboxrun` 配置为调用本项目的 `start.sh`，也会使用同一份项目 `.env`。

## 准备配置文件

首次运行 `./start.sh` 时，如果项目根目录不存在 `config.yaml`，脚本会从 `config.example.yaml` 复制一份。

也可以手动复制：

```bash
cp config.example.yaml config.yaml
$EDITOR config.yaml
```

至少需要确认这些字段：

| 字段 | 说明 |
|------|------|
| `server.host` | 本地监听地址，默认 `127.0.0.1` |
| `server.port` | 本地监听端口，默认 `8080` |
| `providers[].outbound` | 出站协议：`chat_completions` 或 `responses` |
| `providers[].base_url` | 上游服务地址 |
| `providers[].api_key_env` | 从 `.env` 或当前 shell 环境读取的变量名 |
| `providers[].models` | 该 provider 可处理的模型列表 |

## 一键启动

```bash
./start.sh
```

启动脚本会执行以下流程：

1. 准备 `.venv` 虚拟环境。
2. 检查并安装 `requirements.txt` 中的依赖。
3. 缺少 `config.yaml` 时复制 `config.example.yaml`。
4. 加载项目 `.env`。
5. 停止已有实例。
6. 检查端口占用，必要时从 `10000` 起寻找可用端口并写回 `config.yaml`。
7. 后台启动服务并轮询 `/health`。

## 常用命令

```bash
./start.sh                    # 启动服务
./start.sh --log-level debug  # 启动并启用 debug 日志
./start.sh status             # 查看后台进程状态
./start.sh restart            # 重启服务
./start.sh stop               # 停止服务
```

如果需要用持久会话承载服务，可使用：

```bash
tmux new-session -d -s codex_shift_start 'cd /path/to/codex-shift && ./start.sh; exec zsh -l'
```

## 验证服务

启动成功后，脚本会输出监听地址。也可以手动访问：

```bash
curl http://127.0.0.1:8080/health
```

本地控制台地址为：

```text
http://127.0.0.1:8080/admin
```

如果 `start.sh` 因端口占用自动选择了新端口，请使用启动输出中的实际端口。

健康检查响应示例：

```json
{
  "status": "ok",
  "providers": [
    {
      "name": "deepseek",
      "enabled": true,
      "weight": 1.0,
      "outbound": "chat_completions",
      "upstream": "https://api.deepseek.com/v1/chat/completions",
      "models": ["deepseek-v4-pro"],
      "catch_all": false
    }
  ]
}
```

## 日志文件

`start.sh` 默认把服务输出追加到项目根目录的 `.codex-shift.log`。

```bash
tail -f .codex-shift.log
```

运行时 PID 写入 `.codex-shift.pid`，停止服务时会自动清理。
